# AIWiki Roadmap

AIWiki 对外只有一个项目，所以这里写的是能力路线，不是版本分层。

## 1. 更稳的入库体验

- 更顺的 setup 引导
- 更清晰的 doctor 检查
- 更稳定的 Agent handoff
- 更完整的 Obsidian 审阅入口
- 更容易追查失败原因

## 2. 更完整的内容处理能力

- 更好用的资料卡
- 更清楚的 Claim / Evidence 建议
- 更顺手的选题候选
- 更稳定的大纲生成
- 更好复用的模板和输出结构

## 3. 更适合团队和自动化部署

- 更多宿主 Agent 接入方式
- 更适合团队协作的知识库目录
- 更完善的部署说明
- 更清楚的诊断和支持入口
- 更好的服务化落地方式

## 4. 服务层能力

公开软件还是 AIWiki 本身。后续如果要做商业化，重点会放在：

- 部署服务
- 陪跑服务
- 团队集成服务
- 技术支持服务

这些都属于服务层，不是把 AIWiki 拆成多个软件版本。
